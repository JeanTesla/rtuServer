A ->    Captura a entrada de dados pela porta serial definida 
B ->    Ainda será implementado
C ->    Função responsável por responder a requisição retornando um frame para o CLP
D ->    Função responsável por desestruturar o Buffer de entrada (Frame), e retornar um objeto com dados referentes
D.1 ->  Adiciona ao array "pdu" todos os bytes pertencentes ao PDU do frame (Visulizar estrutura em frame_examples/modbus-frame-structure.jpg)
D.2 ->  adiciona ao array "arrayBytes_dataContent" todos os bytes de DADOS vindos do frame. Os dados no frame se encontram entre "Byte count" e "CRC CHK Low"
D.3 ->  Determina em quantos bytes serão divididos os Data Contents
D.4 ->  Iteração para obter os Data Contents separadamente respeitando a quantidade de Bytes por array (definida em D.3)
D.5 ->  Retorno do frame desestruturado em formato de objeto
E ->    Mostra no terminal o dado recebido pela porta serial definida
